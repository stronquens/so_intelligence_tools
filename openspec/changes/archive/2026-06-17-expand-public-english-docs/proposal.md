## Why

The README is now public-facing and written in English, but the `docs/` folder is sparse and still contains Spanish operational notes. Public users need enough English documentation to understand setup, architecture, runtime modes, security, troubleshooting, and the current maturity of each feature.

## What Changes

- Add an English documentation index under `docs/`.
- Add focused English guides for architecture, Linux setup, configuration, local API, desktop UI, selected text correction, screenshot OCR, system audio translation, voice translation virtual microphone, push-to-talk dictation, troubleshooting, security, and the OpenSpec contribution workflow.
- Replace the existing Spanish Linux docs with English content or English compatibility pointers.
- Update README documentation links to point at the new docs index and key guides.

## Capabilities

### New Capabilities

- `public-documentation`: Public documentation covers installation, architecture, features, troubleshooting, security, and contribution workflow in English.

### Modified Capabilities

None.

## Impact

- Affects documentation and OpenSpec artifacts only.
- No runtime behavior, dependencies, API behavior, or service installation behavior changes.
