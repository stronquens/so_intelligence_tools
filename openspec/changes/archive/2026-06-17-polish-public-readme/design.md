## Design

The README should work as the public front door of the repository:

- Hero section with centered logo and concise tagline.
- Badges for Python, platform, local-first and project stage.
- "What it does" section focused on user-visible capabilities.
- OS support matrix with colored round status indicators:
  - green: working/useful now
  - yellow: experimental or partial
  - red: not implemented yet
- Desktop UI screenshot to show the real frontend direction.
- Roadmap split into Now / Next / Later so unfinished work is visible without weakening the project.
- Short installation section for Linux, with links to detailed docs.

Assets:

- Logo: `assets/branding/logo.png`
- Frontend image: `assets/design/overlay-future-reference.jpg`

## Non-Goals

- No new product functionality.
- No generated marketing claims that imply Windows/macOS support exists today.
- No secrets, local `.env` values or private service URLs.
